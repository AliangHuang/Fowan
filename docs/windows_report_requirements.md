# Fowan Windows 汇报工具需求文档

> 文档版本：0.1
> 日期：2026-07-21
> 状态：需求基线；实现前需完成并接受设计提案、组件登记和协议/安全 ADR
> 适用范围：独立 Windows 应用 `Fowan 汇报工具`

---

## 1. 产品定位

`Fowan 汇报工具` 是由 Fowan Toolbox 启动的独立 Windows 工具。用户在前台主动发起生成后，工具从本地待办数据取得指定范围内的已完成和未完成任务，并使用 AI 配置中心为汇报功能选择的默认模型，生成周报、月报或指定时间区间报告。

工具箱只提供入口和启动能力；汇报工具拥有独立窗口、进程、运行状态和后续构建入口。它不嵌入待办主窗口、AI 对话或 AI 配置中心，也不允许在工具内临时选择模型。

首版不做定时生成或自动发送、协作、审批流、云同步、跨设备报表正文历史、数据分析看板和跨工具全文搜索。每次 AI 调用都必须由用户点击“生成”明确触发。

## 3. 信息架构与生成流程

主窗口按以下顺序组织，所有输入在生成前都可修改：

1. **报告范围**：本周、上周、本月、自定义区间，以及与待办一致的清单、层级和日期筛选条件。初始状态没有范围；用户必须先选择范围才能生成。
2. **内容与写作要求**：文本风格、自定义要求、文本模板和填写示例，或文件模板与文件示例。
3. **待办预览**：显示本次会发送的完成/未完成任务数量、时间区间、日期模式和清单/层级筛选摘要；不在此处改写待办数据。
4. **生成结果**：文本模式显示可编辑结果并提供复制；文件模式生成可保存/打开的模板副本。
5. **生成记录**：总览全部进行中、完成、失败和取消的生成尝试；已完成的文本汇报可直接查看，文件汇报显示已保存的输出路径。

当用户点击“生成”时，客户端先固定待办快照与所有输入，完成必要的模型配置、协议和授权检查，再调用 AI。生成期间允许取消；待办随后发生的变化不会影响正在执行的请求。

## 4. 待办筛选与数据范围

汇报工具必须复用 `Fowan.Todo.Shared.Services.TodoQuery` 的筛选结果和语义，不能复制或另行实现日期计算、清单筛选、任务层级筛选或任务完成状态判断。

### 4.1 范围与快捷项

日期区间为闭区间，并使用本机本地日期。

| 快捷项 | 范围定义 |
| --- | --- |
| 本周 | 当前日期所在自然周的周一至周日 |
| 上周 | 当前日期所在自然周之前的完整自然周，即上一个周一至周日 |
| 本月 | 当前日期所在自然月的第一日至最后一日 |
| 自定义区间 | 用户选择的开始日期至结束日期，开始日期不得晚于结束日期，且不等于本周、上周或本月 |

快捷项在日期模式尚未选择时，默认启用“执行周期”；用户已显式选择“开始日期”时，应用快捷项后必须保持“开始日期”模式。上周仅是汇报工具增加的快捷入口，底层筛选语义仍与当前待办一致。

调整筛选只更新用户实际选择的筛选维度，不得把本周、上周或本月自动改为自定义区间。用户手动设定日期后，若日期完整匹配任一快捷范围，必须高亮相应快捷项；只有其他闭区间显示“自定义区间”。“清空筛选”恢复无范围、无显式筛选的初始状态；此时生成操作不可用。

### 4.2 筛选维度与精确语义

筛选界面和待办现有筛选一致，支持：

- 任务清单：全部清单或任一现有清单；
- 任务层级：仅一级、一级和二级、全部层级；
- 日期模式：不按日期筛选、开始日期、执行周期；
- 日期快捷项：本周、上周、本月。

开始日期模式仅匹配 `StartDate` 位于闭区间内的任务。执行周期模式匹配任务执行周期与闭区间相交的任务：开始时间不晚于区间结束，且有效结束时间不早于区间开始。有效结束时间依次取截止日期；没有截止日期的已完成任务取本地完成日期；没有截止日期的未完成任务取当前本地日期；有效结束时间早于开始日期时取开始日期。

汇报始终同时查询完成和未完成任务，并保留 `TodoQuery` 产出的可见层级与排序。软删除/回收站任务不得进入待办预览、AI 请求或报告结果。

### 4.3 发送的任务字段

每项任务以结构化记录发送，包含：标题、备注、所属清单、父子层级、重要标记、开始日期、截止日期、完成状态和完成时间。AI 必须把完成与未完成状态区分呈现；不得把未完成任务表述为已完成，也不得编造待办快照中不存在的工作事实。

## 5. 模板、示例、风格与自定义要求

### 5.1 文本模式

文本模式提供彼此独立的 Notion 式块编辑区：

- 文本模板：支持段落、三级标题、项目符号／编号／待办列表、引用、代码块、分隔线、加粗、斜体、链接和内置表格；用户可通过 `/` 或块左侧 `+` 插入内容块；
- 填写示例：与模板使用相同块编辑、表格粘贴和行列编辑方式，用于展示期望的字段填写方式、语气和篇幅，不作为待办事实来源；
- 自定义要求：本次生成的额外内容、侧重点、禁用表达、格式或写作要求。

表格使用普通 WinUI 单元格和受控块模型，不使用原生 RTF 表格。指针停留在表格边框或单元格时，显示单元格、行、列和整表的轻量操作锚点：可在上下左右插入行列，按当前单元格／行／列／整表清空内容，删除行、列或整张表；表格最少保留一行一列。HTML、RTF 矩形表格和 TSV 可以粘贴；在已有表格中从当前单元格填充并按需扩展。模板和示例均可留空；留空时客户端采用固定的“标题、摘要、已完成表、待推进表”布局。协议传递完整文本内容树（块、可读文本和表格单元格），不传递本地块 ID、Markdown、RTF 或编辑命令；模型返回同类完整内容树，客户端将其验证为可继续编辑的块文档。结果仅以块文档和纯文本复制方式呈现。

### 5.2 文件模式

文件模式只支持 Word `.docx` 和 Excel `.xlsx`。模板文件必填，填写示例文件可选；若提供示例，必须与模板使用相同文件类型。不支持 `.doc`、`.xls`、`.docm`、`.xlsm`、`.xlsb`、PDF 或任何宏文件。

生成时必须先复制模板，之后仅修改该副本，并让用户选择输出位置和文件名。源模板和示例文件不得被覆盖、重命名、删除或就地修改。生成失败、取消或写入校验失败时，不得保留半成品输出文件。

普通模板不要求占位符、标签或预留的“可填充位置”。客户端只读解析 Word 正文段落/表格与 Excel 工作表/单元格的内容树，将样式、公式、图表、图片、宏、页眉页脚、合并单元格、条件格式、数据验证和页面布局作为不可写骨架；模型只返回完整的同类型内容树，不返回 OpenXML、二进制、路径、宏、公式或位置 ID。客户端按树顺序在临时副本中写入段落/表格文字和 Excel 常量，再重新打开并进行形状、列数、内容长度、源模板哈希、OpenXML 可读性和输出类型的强校验。

每次文件候选均从源模板创建新的临时副本。最多进行初始生成加两次修正：校验失败时，仅把不含路径和原文的安全诊断、上轮候选内容发送给模型修正；三轮仍不能通过则删除临时文件并显示可操作错误。Word 新文本继承目标段落或单元格第一个可写 Run 的样式；Excel 保留单元格样式与数值类型，拒绝修改公式或其他受保护对象。只有末尾存在空白样式行、且不依赖公式、合并单元格、条件格式、数据验证或表格范围的普通表格可安全追加行；其他表格必须在既有空间内压缩内容。

### 5.3 文本风格与自定义要求

文本风格固定提供“专业”“通俗”“简洁”三项，并允许同时填写可选的自定义要求。自定义要求是本次生成的独立用户输入，必须与风格、模板、示例和待办快照一起发送给 AI；它可以补充报告内容与形式，但不得覆盖 Core 的安全约束、数据边界、输出结构约束或文件写入保护规则。

自定义要求在本次生成结束、取消或关闭窗口后不持久化，也不得写入日志、诊断包或生成历史。

## 6. AI 配置、协议与职责边界

汇报是独立的 AI 工具功能，标识为 `ai.report`，所需能力为 `ai.report.v1`。AI 配置中心必须新增该功能的默认模型绑定；未配置有效模型时，汇报工具显示明确空状态并提供打开 AI 配置中心的入口，不得使用 `ai.chat` 的绑定或任意其他模型作为回退。

公开协议的 `ai.report.generate` 以结构化字段表达待办快照、筛选摘要、报告类型/时间范围、文本风格、自定义要求和规范化的模板/示例内容树；不得传输本地文件路径、OpenXML、二进制或位置 ID。修正轮可携带上轮候选与安全诊断。结果只能是完整的同类内容树，不能包含可执行代码、宏、公式、任意文件系统操作或 Office 编辑命令。

公开 Fowan 客户端负责窗口、待办查询、输入收集、发送说明、授权交互、取消、内容树解析、候选临时写入、结构强校验、结果编辑和模板副本原子提交。FowanCore 负责模型选择、提示策略、提供方请求、端点授权状态、取消、通用 JSON/大小边界和安全错误归一化，不解析、定位或写入 Office 文件。客户端不得直接调用 AI 提供方、读取密钥、持久化提示词/待办快照，或复制 Core 的编排逻辑。

## 7. 隐私、授权与错误处理

首次向某 AI 端点发送汇报数据前，应用必须展示可取消的明确授权说明，至少列明会发送：筛选后的完整待办详情（含备注）、文本或文件模板、填写示例和自定义要求。授权通过后仍只在用户每次点击“生成”时发送数据。

未配置模型、Core 缺失或不可用、协议不兼容、授权拒绝、输入无效、未选择范围、筛选结果为空、提供方认证/限流/超时/不可用、用户取消和结构化结果校验失败，都必须返回面向用户的安全分类错误；不得展示提供方原始响应正文，不得泄漏密钥、提示词、待办内容、模板内容或自定义要求。文件输出路径仅能在本次生成成功后和本地记录页中显示。

筛选结果为空时不得发送 AI 请求。失败和取消不得修改待办、模板源文件、示例文件或已保存的模板偏好；文件模式仅在完整验证成功后提交输出副本。

## 8. 本地状态与兼容性

首版允许保存用户主动确认保留的模板和填写示例偏好，以及本地生成记录。文本模板与示例保存为 `%LOCALAPPDATA%\Fowan\Report\Preferences` 下的版本化受控 JSON 块文档；只有用户点击保存偏好时才写入，生成结果和本次自定义要求绝不写入这些文件。遗留 `text-template.rtf` 与 `text-example.rtf` 不加载、不转换、不覆盖或删除。生成记录位于 `%LOCALAPPDATA%\Fowan\Report\Records`，包含记录 ID、创建/结束时间、进行中/完成/失败/取消状态、范围、风格、模板类型、任务数量、文件输出状态/路径和标准化安全错误码；已完成的文本记录额外保存其受控块文档，以便在记录页直接查看。支持按时间、状态、范围和输出类型筛选及删除。删除记录会删除其保存的文本结果，但绝不删除输出文件。

除已完成文本记录的受控块文档外，不得保存报告正文、任务快照、模型原始响应、自定义要求或待办内容副本；生成记录中不得保存模板/示例内容。遗留进行中记录在下次启动时标为中断失败。

汇报工具对待办数据只读，必须使用待办 Shared 层提供的路径、加载和查询能力；不得改变 `%LOCALAPPDATA%\Fowan\Todo` 中现有 JSON 格式、位置或待办设置。文本和文件输出仅写入用户主动选择的输出位置。

## 9. 验收与测试

### 9.1 筛选一致性

- 验证本周、上周、自然月和指定闭区间的边界，周以周一至周日计算。
- 验证初始/清空状态无范围、无范围不能生成、筛选调整保持快捷项，以及手动日期精确匹配快捷项或进入自定义区间。
- 验证本周/上周/本月快捷项在未选择模式时启用执行周期，在已选择开始日期模式时保持开始日期模式。
- 验证开始日期与执行周期的差异、无截止日期的已完成/未完成任务、截止日期早于开始日期、清单与层级组合，以及回收站排除。
- 证明汇报查询直接复用 `TodoQuery`，而不是维护第二套筛选逻辑。

### 9.2 AI 与安全

- 验证 `ai.report` 的独立模型绑定、缺失绑定提示、握手能力检查、授权、取消和安全错误映射。
- 验证请求包含完整任务字段、模板/示例与自定义要求，但不包含源文件路径；验证未完成任务不会被错误标记为完成。
- 验证日志、诊断、协议错误和持久化数据不包含密钥、待办内容、模板/示例或自定义要求。
- 验证生成进度按快照、授权、AI、文件选择/写入和终态更新；授权说明及端点文本在不同缩放下完整显示，进行中显示加载动画，成功显示对号、失败显示红叉，文件路径只在副本提交成功后出现。

### 9.3 输出与文件保护

- 验证模板、示例和文本结果都使用块编辑；验证撤销／重做、块插入、纯文本复制，以及表格边框按单元格、行、列和整表执行插入、删除和清空；验证文本请求和结果使用完整内容树、不包含本地块 ID、Markdown、RTF、位置 ID 或编辑命令。
- 验证 HTML、RTF 矩形表格和 TSV 粘贴，表格内从当前单元格填充与扩展；验证旧 RTF 偏好未被加载或删除。
- 验证无占位符 `.docx` 与 `.xlsx` 模板可生成，源模板保持字节不变，输出是独立同格式副本；取消、失败和校验失败不产生半成品。
- 验证首轮成功、两次修正后成功与三轮失败；验证 Word Run 样式继承、Excel 文本/数字常量、公式与非文本对象不变、安全尾部空白行扩展、非安全表格拒绝扩行，以及结构不匹配的安全错误提示。
- 验证生成记录的筛选、文本结果直接查看、文件路径展示、单条删除、确认后的筛选结果批量删除、中断恢复，以及记录删除会清除文本结果但不影响输出文件。

### 9.4 实施门槛

在创建新可执行项目、共享服务、FowanCore 能力、公开协议或任何文件格式处理依赖前，必须完成并接受设计提案，登记公开/私有组件及状态所有者，并为协议、跨仓库、安全/隐私和进程边界变化补充 ADR。实现完成后按仓库规范执行 Debug/Release 构建、测试、架构检查、文档链接检查和 `git diff --check`，且均为零警告、零错误。

## 10. 范围外

- 定时、后台或无人值守的报告生成与发送。
- 邮件、IM、协作、审批、报表共享、云同步和跨设备/跨工具报告正文历史。
- 在汇报工具内手动切换模型、复用聊天默认模型或绕过配置中心。
- 修改待办任务、待办数据格式或待办筛选核心语义。
- 宏、可执行内容、任意公式生成、旧版 Office 文件和源文件就地写入。
